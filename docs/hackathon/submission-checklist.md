# Submission Checklist

## Repo

- [ ] Public GitHub repo
- [ ] Meaningful commit history
- [ ] Root `README.md`
- [ ] Setup instructions
- [ ] Architecture explanation
- [ ] `AI_USAGE.md`
- [ ] `CHANGELOG.md`
- [ ] `DECISIONS.md`
- [ ] `docs/decisions/`
- [ ] `KEEPERHUB_FEEDBACK.md`
- [ ] Contract addresses if deployed

## 0G

- [ ] 0G Storage used for real artifact
- [ ] 0G Compute used or clearly scoped if not
- [ ] working example agent exists
- [ ] explanation of 0G protocol features used
- [ ] architecture diagram or text equivalent

## ENS

- [ ] ENS resolves real agent metadata or policy
- [ ] no identity-critical hard-coded values
- [ ] subname or VC pointer creative use if implemented
- [ ] video shows ENS improving identity/discovery/access

## KeeperHub

- [ ] KeeperHub adapter used meaningfully
- [ ] workflow execution or monitoring shown
- [ ] approach write-up included
- [ ] actionable feedback documented

## Demo

- [ ] 2-4 minutes
- [ ] target under 3 minutes for 0G preference
- [ ] human narration
- [ ] at least 720p
- [ ] no rushed speed-up
- [ ] no text-only music montage
- [ ] shows project in action

## Final submission form

- [ ] project title: ClearIntent
- [ ] short description
- [ ] repo link
- [ ] demo video link
- [ ] live demo link if available
- [ ] team member names
- [ ] Telegram and X contact info
- [ ] partner selections: 0G, ENS, KeeperHub
