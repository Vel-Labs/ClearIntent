# Wallet Demo Strategy

The wallet demo should show ClearIntent as portable agent authority across wallet surfaces.

## Core visual

Show the same agent intent across:

1. ClearIntent human-readable approval preview.
2. Software wallet EIP-712 signing.
3. WalletConnect or mobile wallet signing where available.
4. Hardware-backed signing where available.
5. Audit bundle linking every path to the same intent hash and policy hash.

## Suggested recording sequence

Start with the ClearIntent app and one software wallet. Then progressively split the screen to compare additional wallet surfaces. If hardware wallet display is available, end with a full-screen hardware signing moment so the audience can see the stronger assurance layer.

The intended message:

> ClearIntent meets users where they operate, from accessible software wallets to hardware-backed secure displays.

## Claim boundaries

- Do show ClearIntent's own human-readable metadata.
- Do show EIP-712 typed intent signing where implemented.
- Do show hardware display only if the device path is actually working.
- Do not call a wallet path official Clear Signing unless the vendor approval and tested display path exist.
- Do not imply MetaMask, WalletConnect, Rainbow, Trezor, Tangem, Safe, or any other wallet renders fields unless that exact path has been verified.
