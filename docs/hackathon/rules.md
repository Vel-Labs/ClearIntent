# Hackathon Rules

This file records the rules ClearIntent must satisfy for submission discipline.

## Deadline

Submission deadline: Sunday, May 3, 2026 at 12:00 pm EDT.

Treat this as the real deadline for finalist and partner prize quality.

## Submission requirements

The submission should include:

- project title
- project description
- public GitHub repository
- setup instructions
- contract deployment addresses, if applicable
- demo video
- live demo link, if available
- team member names
- contact info
- partner prize selections
- explanation of partner integrations
- feedback files where required

## Demo video

Required demo video length: 2 to 4 minutes.

Practical target: 2:45 to 3:15, unless a partner specifically requires under 3 minutes. The 0G material says to keep the video under 3 minutes, so target about 2:50.

## Partner selection cap

Select up to 3 partner prizes. If a partner has multiple tracks, one partner selection can still expose the project to multiple tracks under that partner.

ClearIntent target selections:

- 0G
- ENS
- KeeperHub

## Start-fresh rule

Work must begin after the hackathon officially starts to qualify for finalist and partner prizes. Reused public libraries, starter kits, and boilerplates are allowed, but must be transparent.

## Version control

Use meaningful commits. Avoid a single giant commit. The repo history should show progress during the event.

## AI usage

AI tools are allowed, but usage must be documented. Maintain `AI_USAGE.md` and include material prompts or planning artifacts where useful.

## Submission risk checklist

- [ ] Repo is public.
- [ ] README explains setup and architecture.
- [ ] Demo video is between 2 and 4 minutes.
- [ ] Demo video is at least 720p.
- [ ] Demo video uses human narration, not AI voiceover.
- [ ] Partner prize selections are correct.
- [ ] AI usage is disclosed.
- [ ] Reused code and source material are disclosed.
- [ ] Required feedback docs exist.
- [ ] Contract addresses are included if deployed.
