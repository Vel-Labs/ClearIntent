# ERC-7730

Current level: L1 local metadata scaffold.

Current ClearIntent claim: `erc7730-local-metadata`.

Target level: local metadata scaffold, then wallet-specific L4-L6 where supported and tested.

ERC-7730 is the structured metadata direction for clear signing typed or contract data. It can provide wallets with display metadata that raw EIP-712 types alone may not carry.

Clear signing is the broader goal of showing humans meaningful signing details before approval. Ledger has a specific Clear Signing ecosystem and registry, but ClearIntent should treat readable signer display as a wallet-neutral capability that can apply across software wallets, WalletConnect wallets, smart accounts, and hardware-backed signers when supported.

ClearIntent relevance:

- ClearIntent can produce clear-signing-ready metadata for its typed intents.
- Official secure-screen Clear Signing remains wallet and vendor specific.
- Metadata should explain intent meaning without changing the signed payload.
- Generated metadata and validation logs should be stored as audit evidence when available.
- Phase 5B local metadata should later be replaced or supplemented by chain-driven metadata tied to the selected chain/verifying contract context after real wallet validation begins.
- Local metadata generation does not prove wallet acceptance, wallet-rendered preview, secure-device display, or vendor-approved Clear Signing.
- Later chain-driven metadata belongs after Phase 5C signer validation and Phase 2B/3B/4B live-testnet storage, identity, and execution evidence are available.
