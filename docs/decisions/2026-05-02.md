# Decisions: 2026-05-02

## Live ENS binding is policy discovery, not execution approval

- Context: Phase 3B reached `ens-live-bound` for `guardian.agent.clearintent.eth` after live 0G artifact publication and ENS text-record binding.
- Decision: Treat successful live ENS binding as verified identity and policy/audit discovery only. It does not approve execution and must not flip authority approval by itself.
- Consequences: `identity live-status` may report `Identity status: [PASS] ok`, `Live ENS claim: yes`, and `Live 0G claim: bound` while preserving `Authority: blocked`. Execution authority still requires ClearIntent lifecycle verification, human review/signature where applicable, and a verified executor path.

## Phase 4B remains open after 0G/ENS success

- Context: Phase 2B and Phase 3B now provide the live 0G/ENS evidence needed for the KeeperHub demo path.
- Decision: Do not mark Phase 4B complete from 0G/ENS prerequisite evidence. Phase 4B closes only when a selected KeeperHub path records live run or transaction evidence, canonical receipt conversion, and audit persistence.
- Consequences: Roadmap routing moves from ENS binding to KeeperHub execution, with 4B described as unblocked but still open.
