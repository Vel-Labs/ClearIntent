# Decisions: 2026-04-29

## Phase 3A before Phase 2B closeout

- Context: Phase 2B live 0G Storage is readiness-stubbed but blocked on local `.env` values, wallet credentials, live-write opt-in, and funded testnet tokens. ENS identity work can still progress locally because the agent card, resolver interface, text-record mapping, and fixtures do not require live 0G proof.
- Decision: Split Phase 3 into Phase 3A local/mock ENS identity scaffold and Phase 3B live ENS binding. Phase 3A may proceed immediately with `ens-local-fixture` claim level. Phase 3B remains gated on Phase 2B live 0G upload/readback/hash evidence.
- Consequences: ClearIntent can stub the full ENS identity surface now without hard-coding live identity-critical values or overclaiming live provider proof. Live ENS records may reference 0G artifacts only after Phase 2B closeout establishes the exact live 0G claim level.

## Phase 2C compute remains optional

- Context: The required Phase 2 storage path is Phase 2A local artifact semantics followed by Phase 2B live 0G Storage proof. 0G Compute can be useful for risk/reflection, but it is advisory and should not become policy truth or execution authority.
- Decision: Keep Phase 2C as an optional 0G Compute advisory lane, not a required blocker for ENS, KeeperHub, signer, or demo scaffolding.
- Consequences: 2C should be revisited only if it materially improves the hackathon demo or judging story. Any future compute output must be hash-bound, persisted through storage/audit evidence, and presented as advisory rather than authoritative.
