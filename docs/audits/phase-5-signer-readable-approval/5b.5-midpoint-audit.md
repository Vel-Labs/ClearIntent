# Phase 5B.5 Midpoint Audit

## Scope

Reviewed Phase 5B ERC-7730 / Clear Signing Metadata Local Scaffold through the local metadata boundary.

In scope:

- Local ERC-7730-compatible or ERC-7730-inspired metadata generation.
- Display-label mapping from the same field map as the Phase 5A ClearIntent preview.
- Deterministic metadata validation.
- Local claim level `erc7730-local-metadata`.
- Center CLI signer metadata inspection route if implemented by Agent B.

Out of scope:

- Vendor approval.
- Wallet-rendered preview verification.
- Secure-device display verification.
- Chain-driven metadata publication or registry submission.
- Real wallet signing.
- Live provider or onchain behavior.

## Evidence

- `packages/signer-adapter/src/erc7730-metadata.ts` generates local metadata from the Phase 5A approval field map.
- `packages/signer-adapter/src/erc7730-metadata.ts` validates claim level, chain ID, verifying contract, and field ordering against the canonical intent.
- `tests/signer-adapter/signer-adapter.test.ts` proves deterministic metadata generation, metadata hash shape, field-map alignment, explicit limitation strings, and passing local validation.
- `packages/center-cli/src/signer-status.ts` exposes the metadata route while forcing `liveProvider: false` and no wallet/vendor display claims.
- Agent C-owned docs explicitly label metadata as local-only display-support evidence.

## Boundary Review

- Claim level must be limited to `erc7730-local-metadata`.
- Metadata generation does not prove wallet acceptance.
- Metadata validation does not prove wallet rendering.
- Metadata local fixtures do not prove secure-device display.
- ERC-7730 direction does not imply Ledger/vendor approval.
- Later chain-driven metadata belongs after 5C signer validation and the Phase 2B/3B/4B live-testnet evidence path.

## Findings

- Critical: none in Agent C docs scope.
- High: none.
- Medium: chain/verifying-contract context must remain local/test fixture context in 5B and must not be treated as published metadata truth.
- Low: phrase "Clear Signing" can overclaim if not paired with "local metadata only"; docs now preserve that distinction.

## Decision

- pass

Follow-up: revisit chain-driven metadata only after Phase 5C signer validation and the Phase 2B/3B/4B live-testnet path provide validated context.
