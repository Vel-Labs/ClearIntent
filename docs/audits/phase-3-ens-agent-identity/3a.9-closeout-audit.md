## What Is Proven

- Local fixture ENS identity resolves at claim level `ens-local-fixture`.
- `agent.card` resolves through a local URI pointer to typed agent card content.
- `policy.uri`, `policy.hash`, `audit.latest`, and `clearintent.version` are extracted from text records.
- Missing required records block with explicit issue codes.
- Policy hash mismatch blocks with `policy_hash_mismatch`.
- Missing name, missing resolver, unsupported network, and unavailable live lookup are explicit issue states.
- Center CLI exposes `identity status` in human-readable and parse-safe JSON lanes.
- CLI JSON keeps `commandOk: true`, `authorityOk: false`, and `liveProvider: false` for identity status.

## What Remains Unproven

- Live ENS text-record resolution.
- Live ENS-to-0G policy/audit pointer truth.
- Live 0G upload/readback/hash proof.
- ENS-derived identity or policy hash binding into the authority flow.
- Subname role proof.

## Findings

- Critical: none.
- High: none.
- Medium: none.
- Low: `agent.card` uses a local URI and in-repo fixture content. This is correct for 3A but must become config/live-provider backed in 3B before live ENS claims.
