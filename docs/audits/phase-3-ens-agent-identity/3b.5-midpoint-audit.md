# Phase 3B Midpoint Audit

## Scope

Phase 3B midpoint implemented a live ENS read lane behind the existing ENS adapter boundary and Center CLI. This pass did not perform a live ENS lookup because `ENS_PROVIDER_RPC` / `PRIVATE_EVM_RPC_URL` and `ENS_NAME` are not configured in this checkout yet. It does not claim `ens-live-read` or `ens-live-bound` evidence.

## Exact Results

- `npm test -- tests/ens-identity/ens-identity.test.ts`: 13 tests passed.
- `npm test -- tests/center-cli/center-cli.test.ts`: 26 tests passed.
- `npm run validate:center-cli`: passed, including `identity live-status` degraded live-provider readout.
- `npm run typecheck`: passed.
- `npm run clearintent -- identity live-status`: command ok, authority blocked, live provider enabled, claim level `ens-local-fixture`, blocking reason `live_config_missing`.
- `npm run --silent clearintent -- identity live-status --json`: deterministic JSON, `commandOk: true`, `authorityOk: false`, mode `ens-live-read`, live provider `true`, claim level `ens-local-fixture`, blocking reason `live_config_missing`.

## Findings

- Critical: none.
- High: none.
- Medium: live ENS lookup is blocked until an ENS RPC and `ENS_NAME` are configured. No live ENS read claim is made.
- Low: `identity live-status` can reach `ens-live-bound` only when required text records are present and `ENS_EXPECTED_POLICY_HASH` matches `policy.hash`.

## Follow-Up

Once `clearintent.eth` and the selected subname are ready, configure:

```bash
ENS_PROVIDER_RPC=<public or private Ethereum RPC>
ENS_CHAIN_ID=1
ENS_NETWORK=mainnet
ENS_NAME=guardian.agent.clearintent.eth
ENS_EXPECTED_POLICY_HASH=<policy hash if binding is being tested>
```

Then set the required ENS text records and run:

```bash
npm run clearintent -- identity live-status
npm run --silent clearintent -- identity live-status --json
```

Only close Phase 3B after live records are read and the audit records whether the result is `ens-live-read` or `ens-live-bound`.

## Decision

- pass with follow-up

The live ENS adapter and CLI route are ready for operator configuration. Live ENS binding remains open.
