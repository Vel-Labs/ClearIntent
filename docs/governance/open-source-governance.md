# Open Source Governance

ClearIntent should be easy for outside builders to inspect, run, fork, and extend.

## Principles

- Public claims need public artifacts.
- Setup instructions must be honest.
- Reused libraries and AI-generated material must be disclosed.
- Vendor integrations should be modular.
- Security-sensitive limitations should be stated plainly.

## Required public docs

- `README.md`
- `ARCHITECTURE.md`
- `AI_USAGE.md`
- `CONTRIBUTING.md`
- `CHANGELOG.md`
- `DECISIONS.md`
- `docs/hackathon/vendor-tracks.md`
- `docs/hackathon/submission-checklist.md`

## License

Choose a license before the repo becomes public. For hackathon infrastructure, MIT or Apache-2.0 are practical defaults. If patent concerns around SDK-style usage matter, Apache-2.0 is usually cleaner.
