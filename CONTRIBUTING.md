# Contributing

ClearIntent is a strict but fluid hackathon repo. The goal is fast delivery without hidden trust.

## Before contributing

Read:

1. `AGENTS.md`
2. `docs/README.md`
3. `docs/governance/code-quality-standards.md`
4. `docs/governance/worktree-governance.md`
5. the roadmap feature file for your workstream

## Contribution rules

- Keep changes scoped.
- Keep files readable.
- Prefer explicit interfaces over clever coupling.
- Update docs when behavior changes.
- Add audit notes for phase work.
- Do not hard-code demo values that should be resolved from ENS, config, or test fixtures.
- Do not expand scope without recording a decision.

## Pull request checklist

- [ ] Scope is clear.
- [ ] Related roadmap phase is named.
- [ ] Tests or deterministic checks are listed.
- [ ] Docs are updated.
- [ ] Risk/security implications are noted.
- [ ] Changelog entry is added.
- [ ] Decision entry is added if architecture changed.
- [ ] No unrelated files are staged.
