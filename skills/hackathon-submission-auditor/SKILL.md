# Hackathon Submission Auditor

Hackathon submission auditing keeps prize claims honest and tied to working artifacts.

## Read First

- `docs/hackathon/README.md`
- `docs/hackathon/vendor-tracks.md`

- `KEEPERHUB_FEEDBACK.md`
- `AI_USAGE.md`
- `CHANGELOG.md`

## Workflow

1. List each claimed vendor track or prize requirement.
2. Map claims to working code, contracts, docs, demo output, or audit evidence.
3. Flag documented-only or planned work as not submission-ready.
4. Check root feedback files remain present when vendor requirements expect them.
5. Check AI usage disclosure is current.
6. Check unsupported stretch claims are softened or deferred.
7. Produce a pass, pass-with-follow-up, or no-go result.

## Output

Return findings by severity, claim-to-evidence mapping, missing proof, and exact follow-up required before submission.
