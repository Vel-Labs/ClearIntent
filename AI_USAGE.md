# AI Usage

This file records how AI tools are used in the project.

The hackathon rules allow AI assistance, but require transparent documentation of where and how AI tools were used. Treat this as an integrity artifact.

## Tools used

| --- | --- | --- | --- |
| ChatGPT | ideation, governance scaffold, documentation drafting | project owner | project owner |
| Claude | planned coding assistance | project owner / associate | assigned reviewer |
| Cursor / Copilot | planned implementation assistance | assigned developer | assigned reviewer |

## Rules

- AI may draft code, docs, tests, and prompts.
- AI may not be treated as the source of architectural truth.
- Human contributors must review generated code before merge.
- Generated code must obey repo boundaries and quality standards.
- Prompts and planning artifacts that materially shaped the project should be included under `docs/repo-truth/source-material/` or the relevant roadmap package.
